What is HTML?

HTML stands for HyperText Markup Language. It's the standard language for creating and structuring web page content.

HyperText: Refers to hyperlinks that connect web pages, allowing navigation from one page to another (e.g., clicking a link redirects you).

Markup Language: Uses "tags" to add meaning and structure to plain text. Tags "mark up" content to define its role (e.g., headings, paragraphs).

History Insight: The first website ever created (by Tim Berners-Lee in 1990) looked simple: plain text with hyperlinks. No fancy designsâ€”just structure via HTML.

HTML isn't for styling (that's CSS) or interactivity (that's JavaScript). It's purely for content structure.

Important Note: HTML is not a programming language because programming languages have decision-making capabilities (like if-else conditions, loops, and variables). HTML is a markup language used only for structuring content.

Setting Up Your Development Environment

Before writing HTML, set up a code editor. We recommend Visual Studio Code (VS Code) for its simplicity and extensions.

Steps to Install:
  1. Download VS Code: Visit code.visualstudio.com and download for your OS (Windows, macOS, etc.).
  2. Install Extensions: Live Server, Live Preview, Prettier
  3. Create a Project Folder: Open VS Code > File > Open Folder > Create a new folder. Inside, create a file named index.html.
  4. Auto-Save: Enable via File > Auto Save
  5. Previewing: Right-click your HTML file > "Open with Live Server"
Tip: File extension .html tells the system it's an HTML file, just like .mp4 for videos or .jpg for images.

Basic HTML Structure and Tags

HTML uses tags (e.g., <tag>content</tag>) to structure content. Tags can be:

Opening Tag: <tag>
Closing Tag: </tag>
Self-Closing Tag: <tag /> (e.g., for images or lines)

An HTML Element is the full thing: opening tag + content + closing tag (e.g., <p>This is a paragraph.</p>).

1. The Heading Elements (<h1> to <h6>)

What it is: Headings are tags used to define titles and subtitles on your page. They are crucial for creating a logical hierarchy and outline for your content.

Headings create hierarchy and structure, like titles and subtitles. They aren't just for sizeâ€”they give semantic importance (e.g., for SEO and screen readers). <h1> is most important (main title, use once per page); <h6> is least.

<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>

Heading 1

Heading 2

Heading 3

Heading 4

Heading 5
Heading 6

The Purpose: To structure your document in a way that is understandable to both humans and machines (like search engines and screen readers). It is not just for making text big; it is for giving the text structural importance.

Best Practice: Don't skip levels (e.g., <h1> then <h2>, not directly to <h3>). Helps SEO by prioritizing content for search engines.

2. The Paragraph Element (<p>)

What it is: The paragraph tag is the most common tag you'll use. It's for grouping sentences and blocks of text together.

For blocks of text. Browsers add automatic spacing before/after. Use multiple <p> tags instead of manual line breaks for better structure.

<p>This is the first paragraph. It contains several sentences about a topic.</p>

<p>This is a second, separate paragraph. The browser will display it with a space between it and the first one.</p>

This is the first paragraph. It contains several sentences about a topic.

This is a second, separate paragraph. The browser will display it with a space between it and the first one.


The Purpose: To define a distinct paragraph of text. Browsers automatically add a little bit of space before and after a <p> element, separating it from other content. You should not use multiple line breaks; you should use multiple <p> tags.

Tip: For random placeholder text, type lorem50 in VS Code (with Emmet enabled) to generate 50 words.

3. Horizontal Rules (<hr>)

Creates a thematic break (horizontal line) to separate content. Indicates a change in topic. Self-closing tag: Use <hr /> (recommended).

<p>Topic 1</p>
<hr />
<p>Topic 2</p>

Topic 1

Topic 2

4. Line Breaks (<br>)

Forces a new line without starting a new paragraph. Useful for addresses or poems. Self-closing tag: Use <br />.

<p>221B Baker Street<br />London, England</p>

221B Baker Street
London, England

5. Lists (<ul>, <ol>, <li>)

For grouping related items. Lists involve nesting: <li> items inside <ul> or <ol>. Unordered List (<ul>): Bulleted (dots). Ordered List (<ol>): Numbered. Purpose: Structure related content (e.g., ingredients or steps).

<ul>
  <li>Tea Bag</li>
  <li>Water</li>
  <li>Sugar</li>
  <li>Milk</li>
</ul>

<ol>
  <li>Boil water.</li>
  <li>Add tea bag, sugar, and milk.</li>
  <li>Boil for 5 minutes.</li>
  <li>Serve in a cup.</li>
</ol>

  • Water

  • Milk

  1. Boil water.
  2. Add tea bag, sugar, and milk.
  3. Boil for 5 minutes.
  4. Serve in a cup.

6. Anchor Tags (<a>)

Creates hyperlinks for navigation. Makes text clickable to link to other pages/files. Key Attribute: href (URL destination). Target Attribute controls where link opens: _self (default): Same tab, _blank: New tab.

<a href="https://www.coderarmy.in/">Visit Coder Army</a>

<a href="https://www.coderarmy.in/" target="_blank">Visit in New Tab</a> 		Visit Google

Visit in New Tab

7. Image Elements (<img>)

Embeds images. Self-closing tag. Key Attributes: src (Image URL/path - required), alt (Alternative text for accessibility), height and width (Size in pixels - optional). Tip: Copy image URLs by right-clicking > "Open Image in New Tab" > Copy address.

<img src="https://cdn.britannica.com/16/234216-050-C66F8665/beagle-hound-dog.jpg" alt="A cute dog" height="100" width="100" />

Comments in HTML

Use <!-- comment --> for notes that don't show in output. Shortcut: Ctrl+/ (Windows) or Cmd+/ (Mac) in VS Code.

<!-- This is a comment, not visible on the page -->

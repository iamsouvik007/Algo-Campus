File Path System

The First Principle: A website is not one single file. It's a collection of different files (HTML, CSS, JavaScript, images, videos, fonts) that all need to work together.

The Core Problem: If your index.html file needs to display an image named logo.png, how does the HTML file tell the browser where to find logo.png?

The browser needs an exact, unambiguous address to locate the file. A file path is that address.

Relative File Paths (The Most Important)

A relative file path gives directions to a file starting from the location of the file you are currently in. You will use this 99% of the time for linking your own files together.

Example Project Structure:

my-website/
├── index.html
├── about.html
│
├── images/
│   ├── logo.png
│   └── hero.jpg
│
└── pages/
    ├── contact.html
    └── terms.html

Scenario 1: Same Folder
From index.html to about.html: Just use the filename
<a href="about.html">About Us</a>

Scenario 2: Sub-folder (Going Down)
From index.html to logo.png in images folder:
<img src="images/logo.png" alt="Logo">

Scenario 3: Parent Folder (Going Up)
From contact.html to logo.png:
<img src="../images/logo.png" alt="Logo">
../ means "go up one level"

Absolute File Paths

An absolute file path gives the full, complete URL to a resource on the web. It starts with http:// or https://.

When to use it: ONLY when linking to resources that are NOT on your own website.

Examples:

<!-- Linking to external website -->
<a href="https://www.google.com">Search on Google</a> 		Search on Google

Critical Warning: Never use absolute paths for your own files (like C:/Users/...). This only works on your computer and will break when uploaded to a server.

Why: Browsers run your code in a secure sandbox and do not allow websites to access your computer's file system. A path like C:/Users/You/Pictures/logo.png will not be readable on the web and also only exists on your machine.

Security Example: If browsers allowed local paths, a malicious site could try:
<!-- Dangerous if allowed (it is NOT allowed) -->
<img src="C:/Users/You/Documents/Passwords.txt" alt="leak" />

Always prefer relative paths for your own assets.

Operating System File Paths

Windows Absolute Path:
• Starts with drive letter (C:, D:)
• Uses backslash (\) as separator
• Example: C:\Users\JohnDoe\Documents\report.docx

macOS/Linux Absolute Path:
• Starts with single forward slash (/)
• Uses forward slash (/) as separator
• Example: /Users/johndoe/Documents/report.docx

HTML Boilerplate Code

The First Principle: A web browser needs specific, predictable instructions to do its job. It cannot guess your intentions.

The Core Problem: We need to give the browser essential "setup information" before it starts rendering our visible content.

The Modern HTML5 Boilerplate:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>

</body>
</html>

Breaking Down the Boilerplate

1. <!DOCTYPE html>
• Document Type Declaration - must be first line
• Tells browser to use modern HTML5 standards
• Prevents "quirks mode"

2. <html lang="en">
• Root element that wraps entire page
• lang="en" declares English as primary language
• Important for SEO and accessibility

3. <head>
• Container for metadata (not visible on page)
• Contains information for browser, not users

4. <meta charset="UTF-8">
• Sets character encoding to UTF-8
• Universal standard for web text
• Prevents text encoding issues

5. <meta name="viewport"...>
• Essential for mobile responsiveness
• width=device-width: matches screen width
• initial-scale=1.0: sets 100% zoom level

6. <title>
• Text shown in browser tab
• Used in bookmarks and search results

7. <body>
• Container for all visible content
• Where all your content tags go

Why Websites Work Without Boilerplate

Browsers are Extremely Forgiving: They guess missing elements to display something rather than crash.

What Happens When You Skip Boilerplate:
• Browser enters "Quirks Mode" (unpredictable rendering)
• Missing tags are implicitly generated
• Character encoding is guessed (can break special characters)
• Filename used as default title

Why Use Boilerplate:
• Consistent rendering across all browsers
• Proper mobile responsiveness
• Better SEO and accessibility
• Professional, future-proof development

The <div> Element (Generic Container)

The Problem: How do you group different elements together as a single unit for styling or layout?

The Solution: <div> (division) - a generic, meaningless container that groups other elements.

Example:

<div>
    <img src="avatar.png">
    <h2>Arjun Kumar</h2>
    <p>Loves to code and teach HTML.</p>
</div>

Arjun Kumar

Loves to code and teach HTML.

Now you can style the entire group as one unit with CSS.

class and id Attributes

The Problem: How do you identify and target specific elements or groups of elements?

Two Types of Labels:

id - Unique Identifier
• Like a Social Security Number
• Only ONE element per page can have specific id
• Used for unique elements (main navigation, search form)
• Targeted with # in CSS

class - Reusable Classifier
• Like a job title (many people can be "Engineer")
• Multiple elements can share same class
• Used for similar styling or function
• Targeted with . in CSS

<!-- Unique main header -->
<div id="main-header">...</div>

<!-- Special profile card -->
<div id="featured-profile" class="profile-card">...</div>

<!-- Standard profile cards -->
<div class="profile-card">...</div>
<div class="profile-card">...</div>

CSS Targeting:
• #main-header { ... } (targets unique ID)
• .profile-card { ... } (targets ALL elements with this class)
• #featured-profile { ... } (targets one special element)

The <span> Element (Inline Container)

The Problem: How do you style a piece of content within a line of text without creating a line break?

The Solution: <span> - an inline container that doesn't break the text flow.

Analogy: Like using a highlighter pen on specific words in a book.

<p>This is very <span class="highlight">important</span> information.</p>

This is very important information.

Key Difference:
• <div> creates a block (new line)
• <span> stays inline (no line break)

Semantic HTML Elements

Modern HTML provides meaningful alternatives to generic <div> elements:

<header>
• Container for introductory content
• Usually contains logo, navigation, main heading
• Can be used for page header or section header

<footer>
• Container for closing content
• Usually contains copyright, contact info, secondary links
• Appears at bottom of page or section

<main>
• Most important element for content structure
• Contains the main, unique content of the page
• Should NOT contain repeated elements (header/footer)
• Only ONE <main> tag per page

<nav>
• Container for navigation links
• Helps screen readers identify navigation areas

<section>
• Generic section of content with a theme
• Usually includes a heading

<article>
• Self-contained content (blog post, news article)
• Should make sense when taken out of context

Multi-page Website Structure

The Fundamental Truth: Complex information is broken into distinct, self-contained topics, just like book chapters.

The Core Problem: How do we create distinct pages and provide seamless navigation between them?

The Solution:
1. Consistent Structure: All pages share common look (header, navigation, footer)
2. Navigation System: Links using <a> tags on every page

Example Navigation:

<nav>
    <a href="index.html">Home</a> |
    <a href="about.html">About</a> |
    <a href="services.html">Services</a> |
    <a href="contact.html">Contact</a>
</nav>

Best Practice: Include the same navigation on every page to create a cohesive website experience.

Summary - Rules of Thumb

File Paths:
• For your own files: ALWAYS use relative paths
• For external resources: ALWAYS use absolute paths (https://)
• Never use computer-specific absolute paths (C:\...)

HTML Structure:
• ALWAYS start with proper HTML5 boilerplate
• Use semantic elements (<header>, <main>, <footer>) when possible
• Use <div> and <span> for generic grouping
• Use class for styling multiple elements
• Use id for unique elements only

Multi-page Websites:
• Keep consistent navigation on all pages
• Use relative paths for internal links
• Maintain consistent header/footer structure
